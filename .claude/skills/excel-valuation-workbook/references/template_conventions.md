# Val-Studio 워크북 규약 (자체 시트 아키텍처 — W0 정본)

참고 모델·타사 양식을 **복제하지 않고**, 모델링 기초 지식으로 정제한 자체 아키텍처. 시트명·
레이아웃·색상은 이 문서를 따른다.

---

## 1. 시트 아키텍처 (점진 성장)

처음부터 멀티시트를 찍지 않는다. 각 단계가 자기 시트를 만들며 워크북이 자란다.

| 단계 | 시트 | 역할 |
|------|------|------|
| W0 | `DCF` | 스파인(가정 블록 + 5개년 + 결과), scaffold 생성 |
| W0 | `_VS_STATE` | 상태·가정 대장·매핑 대장(숨김) |
| W1 | `Research` | Company Brief(숫자 셀 + 서사 텍스트) = 리서치 SSOT |
| W2 | `FS_Hist` | `Raw`(원문 불변)/`Normalized`/`Map` 3영역 |
| W3 | `Reclass` | 계정 태깅 + `_A`(실사조정)/`_F`(최종) 레이어 |
| W4 | `Fcst_Rev`·`Fcst_Cost`·`Capex_Dep`·`WC` | 드라이버별 추정 |
| W5 | `WACC` | CAPM 빌드업 |
| W7 | `Scenario` | Up/Base/Down |
| W8 | `Sens` | WACC×PGR 5×5 |

**참조 방향 — 셀 단위 비순환(DAG)**: 기본은 `뒤 시트 → 앞 시트`. 다만 판정 단위는
**시트가 아니라 셀**이다. 시트 A·B 가 서로를 참조해도 **참조되는 셀 집합이 분리**되면 허용한다.

> 실측(모델러스 5.4 §1.1): `Scenario!I15 = EBIT!I37`(과거 실적) 이면서
> `EBIT!O37 = Scenario!O15`(추정 드라이버) — **시트 레벨로는 순환이지만 과거열/추정열이
> 서로 다른 셀이라 셀 레벨로는 DAG**. 시트 단위로만 읽으면 이 정당한 패턴을 금지하게 된다.

**셀 단위 순환은 금지**한다(반복계산 의존). 단 3표 완결(BS 대차·CF 연결) 목적의 이자 순환만
예외이며, 그때는 `IF($순환스위치="ON", 스케줄, 0)` 형태의 **명시적 차단기**를 두고 그 셀을
가시 상태 헤더에 노출한다. **FCFF DCF 는 무차입 기준이라 순환이 원천적으로 불필요** —
DCF 스파인에는 어떤 경우에도 도입하지 않는다.

**원자료 격리 (`r` 접두)**: 붙여넣은 외부자료(재무제표 원문·peer 시장데이터·거시 통계·
세그먼트)는 `r` 접두 시트에만 두고 **절대 수정하지 않는다**(`rFS`·`rPeer`·`rMacro`·`rSegment`).
모델 시트는 오직 참조만 한다. 정규화·조정이 필요하면 **별도 시트에서 수식으로** 하고 원본은
남긴다 — 원본을 덮어쓰면 tie-out 의 기준점이 사라진다. `FS_Hist` 의 `Raw` 영역이 같은 원리의
시트 내 구현이며, peer·거시로도 동일하게 일반화한다.

## 2. 색상 규약 (3색 + 강조)

| 색 | 의미 | 예 |
|----|------|-----|
| **Blue** | 직접 입력 hard 값 | 가정값, 원천 데이터 |
| **Black** | 해당 시트 계산 수식 | EBIT=매출−원가−판관비 |
| **Green** | 타 시트 참조 값 | `=Research!C10` |
| **Yellow fill** | 핵심 가정 강조 | WACC·PGR |

**hard number 1곳 규율**: 하드값은 최초 1곳만 입력, 나머지는 참조(soft). 상류 시트가 생기면
DCF 가정 셀(Blue)을 상류 참조 수식(Green)으로 승격 — 교체 전후 per_share 불변을 `roundtrip.py`로 확인.

## 3. 수식·서식 규약 (anthropic xlsx 규약 채택)

- **살아있는 수식**: 계산 결과는 하드값이 아니라 수식으로(감사 추적). `scaffold.py`가 이미 이렇게 생성.
- **함수 화이트리스트**: Excel-2007 세대 우선. post-2007(XLOOKUP·FILTER·SORT·LET·LAMBDA)은 호환성 위험 → 피하고, 불가피하면 `_xlfn.` 접두 확인.
- **서식**: 백분율은 fraction으로 저장(0.113, 표시만 %), 0→'-', 음수는 괄호.
- **외부링크 금지**: 타 파일 참조(`[Book2.xlsx]`)는 `#REF!` 위험 → 값으로 붙여넣기.
- **recalc 게이트**: 산출 xlsx는 LibreOffice headless recalc 0에러 확인 후 출고(W8 외곽 셀 검증).

## 4. DCF 스파인 레이아웃 (scaffold.py 생성 — 참조용)

`scaffold.py`가 만드는 `DCF` 시트 고정 레이아웃(`roundtrip.py` 왕복 대상):

```
가정 블록:  C3=WACC  C4=g  C5=주식수  C6=비영업자산  C7=순차입부채
연도(C..G): 10=Year 11=매출 12=매출원가 13=매출총이익 14=판관비 15=EBIT
            16=법인세 17=NOPLAT 18=D&A 19=CAPEX 20=ΔNWC 21=FCFF 22=할인기간 23=현가계수 24=PV
결과:       C27=명시적PV합 C28=TerminalFCFF C29=TV C30=TerminalPV C31=EV C32=주식가치 C33=주당가치
메타:       C37=effective_tax_rate C38=terminal_fcff_override C39=terminal_reinvestment_rate
```

이 레이아웃을 임의로 바꾸면 `roundtrip.py` import가 깨진다 — 확장은 build 도구와 함께.

## 4.5 상시 검증·조회 규약

### 가시 상태 헤더 (전 시트 I1:J3)
모든 시트 상단 우측에 전역 상태를 노출한다 — `Scenario` / `Target Price` / `Stage`.
`_VS_STATE` 는 숨김 시트라 사람이 못 본다. 어느 시트를 열든 "지금 이 모델이 어떤
상태인지"가 보여야 한다(모델러스 정본은 전 시트에 Circuit·Scenario·Target Price 노출).

### CHECK 행 — **허용오차 필수**
합보존·정합을 워크북에서 **상시** 확인한다. 게이트 스크립트는 인제스트 시점 1회지만,
CHECK 행은 편집 중에도 살아있다.

```
=IF(ABS(좌변-우변) < 0.001, "TRUE", 좌변-우변)
```

- **불일치 시 TRUE/FALSE 가 아니라 *잔차 금액*을 표시** — 어디가 얼마나 틀렸는지 바로 추적.
- ⚠️ **정확일치 비교(`=IF(A=B,…)`) 금지**. 부동소수 노이즈로 경제적으로 완전히 일치하는
  연도가 FALSE 로 뜬다(모델러스 5.4 §4 D1 실측: 잔차 `-7.1e-14` 로 2개 연도 오작동).
- 허용오차 SSOT = `template_schema.CHECK_TOL`(백만원 단위 0.001 = 1천원).

적용 위치: `FS_Disagg` 세분합=원계정 · `Fcst_Cost` 배분합=총액 · (3표 확장 시) 대차·현금연결.

### 원자료 조회 — 2차원 INDEX/MATCH
표에서 값을 가져올 땐 직접 셀참조 대신 **행=키, 열=필드명** 이중 MATCH 를 쓴다.

```
=INDEX($B$6:$J$11, MATCH($B17,$C$6:$C$11,0), MATCH(C$16,$B$6:$J$6,0))
```

열 순서가 바뀌어도 깨지지 않고, 원본 수정이 자동 반영된다(손으로 옮겨적기 금지).
⚠️ **헤더 문자열이 곧 조회 키** — 참조하는 쪽 헤더가 원본 헤더와 정확히 같아야 한다
(`"회사(①조회)"` 같은 장식을 붙이면 `#N/A`).
