## 0. 핵심 메타포 — "프로젝트 = 워크북, 화면 = 시트"

실무자의 멘탈 모델은 이미 엑셀 워크북이다(리포트 정본: 개요·회사소개·주요가정·
시나리오·DCF·매출상세·FA·WC·유사회사·WACC 시트). 앱의 정보 구조를 이 메타포로
계승하면 학습 비용이 0에 수렴한다:

| 엑셀 | 앱 |
|---|---|
| 워크북 파일 | **프로젝트**(밸류에이션 용역 1건) |
| 시트 그룹(Assumption/Results/Backdata) | **LNB 의 워크플로우 단계** |
| 개별 시트 | **하단 시트탭**(단계 내 상세 화면) |
| 셀 컬러코딩(Blue입력/Black계산/Green참조) | 필드 상태 표시(입력가능/계산값/참조) |
| Ctrl+[ 참조 추적 | 참조 클릭 → 가정 화면으로 점프 |

## 1. 계층 구조 (3층)

```
L0  홈 = 프로젝트 목록 (최초화면)
     └ 새 프로젝트(모드 선택: 평가인/감사인) · 기존 열기
L1  프로젝트 워크스페이스 (모드별 셸: 헤더+LNB+본문+시트탭+우측패널)
L2  화면(시트) 내 패널·테이블
```

### 최초화면 판단 — 랜딩 페이지는 만들지 않는다
로컬 도구이므로 마케팅성 정적 페이지 불요(사용자 우려 정합). **홈 = SharePoint
벤치마크의 극미니멀 프로젝트 리스트**(아이콘·프로젝트명·모드 뱃지·수정시각·대상회사)
하나. 빈 상태면 "새 평가 시작" CTA 1개만. 설정(BYOK)은 홈·헤더 어디서든 진입하는
모달/화면 1개.

## 2. 모드(평가인/감사인) — 전환이 아니라 프로젝트 속성

사용자 판단대로 역할은 용역 단위로 고정(중간 전환 거의 無). 따라서:
- **모드는 프로젝트 생성 시 1회 선택**하는 속성 — 이후 셸의 LNB·화면 세트가 통째로
  달라진다. 앱 어디에도 "모드 전환 스위치"를 두지 않는다(상태 혼선 원천 차단).
- 역할이 바뀌면 = **다른 프로젝트를 새로 만든다**(같은 대상회사라도 별건). 헤더에
  모드 뱃지(평가인/감사인)만 상시 표시.
- 부수 효과: 감사인 트랙의 독립성 원칙(평가자 결론 재사용 금지)이 데이터 격리로
  구조 보장됨.

## 3. 축별 배치 (5축)

```
┌────────────────────────────────────────────────────────────┐
│ 헤더: 로고 │ 프로젝트명 · 모드뱃지 │ (우) 게이트 요약 · BYOK 상태 │
├──────┬─────────────────────────────────────────┬───────────┤
│ LNB  │  본문: 상단 KPI 행(해당 시) + 고밀도 패널    │ 우측 패널  │
│ 단계  │                                         │ (접이식)   │
│ 순차  │                                         │ 근거·AI    │
├──────┴─────────────────────────────────────────┴───────────┤
│ 하단 시트탭: 현 단계의 상세 시트들                              │
└────────────────────────────────────────────────────────────┘
```

- **헤더(상단)**: 정체성+상태만. 게이트 요약(FAIL n·WARN n — 클릭 시 해당 finding
  으로 이동), BYOK 연결 상태 점. 액션 버튼은 두지 않는다(액션은 본문 소유).
- **LNB(좌)** = **워크플로우 단계 축**(장 단위). 각 항목에 진행 상태 표기:
  ✓완료 · ●현재 · ○미착수. **잠금은 하지 않는다** — 자유 이동 허용하되, 선행 단계
  미완이면 화면 상단에 게이트 경고 배너(엑셀도 시트 이동은 자유, 대신 검증이 잡는
  구조와 동형).
- **하단 시트탭** = **단계 내 시트 축**. ⚠️ 현 구현 교정: 지금은 LNB 와 시트탭이
  같은 목록의 중복 — 역할 분리로 수정한다(LNB=장, 탭=시트).
- **우측 패널(접이식, 기본 접힘)** = **컨텍스트/판단보조 축**: 선택한 필드·행의
  ①provenance(출처 문서·셀·기간) ②관련 audit finding 상세 ③**AI 제안**(북 근거
  인용·uncertain ⚖️ 큐). LLM 은 본문에 직접 쓰지 않고 이 패널에서만 제안 —
  "판단 보조" 원칙의 공간적 격리. 채택 시에만 본문(입력 필드)에 반영.
- **대시보드는 별도 화면으로 만들지 않는다**: 전역 현황 = 홈 리스트, 프로젝트 현황 =
  0번째 시트 "개요(Cover)"(KPI·게이트 현황·다음 할 일) — 워크북의 Cover 시트 관례.

## 5. 엑셀 원칙의 UI 번역 (필드 수준 UX)

- **단방향 데이터 흐름**: 입력은 "2.가정"에만 존재(hard number 1곳). 4.밸류에이션
  화면의 모든 숫자는 읽기전용 계산값 — 수정하려면 참조 클릭 → 가정으로 점프
  (Ctrl+[ 의 UI 화). 순환·중복 입력 원천 차단.
- **필드 3상태 표기**(컬러코딩 계승, 단 채도 절제): 입력가능(테두리 강조)/계산값
  (회색 배경 읽기전용)/타 화면 참조(링크 스타일). 브랜드색은 여기 쓰지 않는다.
- **가정마다 근거 한 줄 슬롯**: 참고 모델 §F1 정본(숫자+정성 근거 병기) — 값 옆에
  근거 텍스트(출처·판단), 비면 audit 이 WARN. 우측 패널의 provenance 와 연결.
- **모든 결과 화면은 KPI+게이트 동반**(현 DCF 화면 문법 유지) — 숫자만 단독 노출 금지.
